**US-05: Universal Week 1 Milestones (All Roles)**
> As a new hire of any role, I want to see the mandatory setup tasks for Week 1 so that I complete all essential onboarding steps.

**1. Business Context**
The first week of onboarding consists of universal foundational tasks that every new employee must complete regardless of their role. These include IT setup, security compliance, and basic communication tool access. Streamlining these into clear milestones ensures no one falls behind on baseline requirements.

**2. In Scope**
- Defining and displaying a set of mandatory milestones for Week 1.
- Applying these milestones to all new users across Engineering, Product, and Design roles.
- The specific checklist items: Everest email sign-in, 2FA setup, Slack channel join, Zoom setup/profile picture, and mandatory security training.

**3. Out of Scope**
- Role-specific training tasks for Week 1 (these are handled in separate stories).
- Automated verification of task completion (e.g., we do not use an API to check if they *actually* joined Slack; we rely on the user checking the milestone).

**4. Assumptions/Dependencies**
- *Dependency:* The user has successfully authenticated and reached the Journey view.
- *Assumption:* Links to external tools (like the security training portal or Slack workspace) are readily available to embed in the milestone descriptions.

**5. Acceptance Criteria - Given When Then**

*Scenario 1: Viewing Week 1 Universal Tasks*
**Given** I am a new hire who has just logged in and selected my role
**When** I view the "Week 1: Foundations" section of my onboarding journey
**Then** I must see the following mandatory milestones listed:
  - Sign in with Everest email + reset temporary password
  - Enable 2-factor authentication
  - Join Everest Slack channel
  - Set up Zoom with Everest email + update profile picture
  - Complete mandatory security training

*Scenario 2: Cross-Role Consistency*
**Given** I am a new hire logging in as a Designer, Engineer, or Product Manager
**When** I view my Week 1 milestones
**Then** the exact same set of foundational IT and security milestones described in Scenario 1 must be present, regardless of my selected role.

*Scenario 3: Milestone Interaction*
**Given** I am viewing a universal Week 1 milestone
**When** I have completed the required action (e.g., joined Slack)
**Then** I can interact with the milestone's completion indicator to mark it as done, updating my overall progress.
